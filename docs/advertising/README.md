## 🎯 部门职责

广告部门是交易系统**唯一的对外通知出口**。所有 Agent 完成操作后必须通知本部门，由本部门统一排版美化后通过飞书卡片推送给用户。

**核心使命：美观、准确、及时的对外通知。**

## 📋 工作流程

```
Agent 完成操作
    │
    ▼
通知 ADV-001（携带操作结果/状态变更）
    │
    ▼
去重检查（读取 /tmp/hermes_ad_last.json）
    │
    ├── 命中去重规则 → 跳过，不推送
    │
    └── 未命中 → 排版美化 → send-card.ts → 飞书推送给用户 → 更新缓存
```

---

## 🚫 去重规则

发送前**必须先检查**去重缓存 `/tmp/hermes_ad_last.json`。

### 跳过条件（满足任一即跳过）

| # | 条件 | 说明 |
|---|------|------|
| 1 | 同一 Agent + 同一股票 + 同一结论 | 如连续两次 HOLD 同一只股票 |
| 2 | 同一 Agent + 同一股票 + 价格波动 < 0.5% | 价格几乎没变 |
| 3 | 同一 Agent + 距离上次通知 < 10 分钟 | 避免刷屏 |
| 4 | 系统状态无变化 | running 数量等无变化 |

### 必须发送（无论去重）

| 场景 | 说明 |
|------|------|
| 🔴 交易成交 | BUY/SELL 执行完成 |
| 🔴 Agent 状态变更 | 崩溃/复活/淘汰 |
| 🔴 投票结果 | 选举通过/驳回 |
| 🔴 熔断/紧急事件 | 系统级异常 |
| 🟡 距离上次同类通知 > 30 分钟 | 定期更新 |

### 缓存格式

```json
{
  "agent": "AGT-004",
  "symbol": "NVDA.US",
  "verdict": "HOLD",
  "price": 216.44,
  "time": "2026-05-23T10:00:00"
}
```

## 🎨 飞书卡片规范

通过 `send-card.ts` 发送，Agent 自行设计 card JSON，send-card 纯透传。

### 颜色编码

| 颜色 | 含义 | 使用场景 |
|------|------|----------|
| 🟢 绿色 | 交易/盈利 | BUY/SELL 成交、止盈、盈利报告 |
| 🔵 蓝色 | 状态/日常 | 系统状态、例行扫描、持仓更新 |
| 🟠 橙色 | 警告/影子期 | 风控预警、Agent 进入影子期 |
| 🔴 红色 | 熔断/淘汰 | 熔断触发、Agent 淘汰、系统崩溃 |
| 🟣 紫色 | 选举 | 投票发起、结果公布 |
